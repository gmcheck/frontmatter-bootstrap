# Frontmatter Bootstrap 插件详细说明文档

## 概述

**Frontmatter Bootstrap** 是一个 Obsidian 插件，专注于为你的笔记提供长期稳定的 metadata 基础设施。

### 核心定位

这不是一个 AI 插件、知识管理系统或自动分类工具。它是一个 **极简、稳定、低侵入** 的 metadata 基础设施插件。

### 设计哲学

- **极简** — 只做必要的事情
- **长期稳定** — 不追逐短期功能
- **低侵入** — 不干扰用户工作流
- **可预测** — 行为明确，无意外
- **确定性** — 不做语义理解或智能分析

---

## 功能模块

### 功能一：自动插入 Frontmatter 模板

当创建新的 Markdown 笔记时，自动插入统一的 YAML frontmatter 模板。

#### 触发条件

同时满足以下条件时才会插入模板：

1. 文件是 Markdown 文件（`.md` 扩展名）
2. 文件内容为空（无任何字符）
3. 文件没有已有的 frontmatter

> 三个条件缺一不可。已有内容的文件、非空文件、已有 frontmatter 的文件都不会被修改。

#### 默认模板

```yaml
---
type: note
domain:
status: active
created: 2026-05-09
updated: 2026-05-09
source: personal
tags:
---
```

#### 字段说明

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `type` | Select | 笔记类型 | `note` |
| `domain` | Text | 领域分类（用户自行填写） | 空 |
| `status` | Select | 状态 | `active` |
| `created` | Date | 创建日期（YYYY-MM-DD） | 当天日期 |
| `updated` | Date | 更新日期（YYYY-MM-DD） | 当天日期 |
| `source` | Select | 来源 | `personal` |
| `tags` | Tags | 标签 | 空数组 |

#### Vocabulary 系统

**type、status、source** 字段使用 **Vocabulary 系统**，用户可以自定义允许值：

**默认 type vocabulary：**
| 值 | 说明 |
|------|------|
| `note` | 普通知识笔记 |
| `project` | 具有生命周期的项目型内容 |

**默认 status vocabulary：**
| 值 | 说明 |
|------|------|
| `active` | 当前活跃维护中 |
| `done` | 已完成，但仍保留在活跃知识空间 |
| `archived` | 已退出活跃知识空间 |

**默认 source vocabulary：**
| 值 | 说明 |
|------|------|
| `personal` | 以个人思考/整理为主 |
| `external` | 以外部资料整理为主 |

> 用户可以在设置页面中添加、删除、修改 vocabulary 值。

**domain**（预设选项，可扩展）
- `music`
- `bigdata`
- `aigc`
- `ecommerce`
- `programming`
- `finance`
- `system`

> domain 字段是自由文本，预设值仅用于健康检查时的参考。你可以填写任意值。

---

### 功能二：自动更新 updated 字段

当 Markdown 文件保存时，如果文件包含 `updated` 字段，插件会自动将其更新为当前日期。

#### 工作原理

1. 监听文件修改事件
2. 检查文件是否有 `updated` 字段
3. 如果有，更新为当前日期（YYYY-MM-DD 格式）

#### 安全保证

- **仅更新 `updated` 字段** — 不会修改 `created` 或其他字段
- **不修改正文内容** — 只操作 frontmatter
- **不覆盖已有 frontmatter** — 使用 Obsidian 官方 API 安全操作
- **不添加字段** — 如果文件没有 `updated` 字段，不会自动添加

#### 防抖机制

插件内置 500ms 防抖，避免频繁保存时重复更新。同一文件在 500ms 内只会触发一次更新。

### 功能三：Metadata Health Check

提供手动执行的 metadata 健康检查功能，帮助你发现笔记中的 metadata 问题。

#### 使用方法

1. 打开命令面板（`Ctrl/Cmd + P`）
2. 输入 `Metadata Health Check`
3. 选择检查范围：
   - **Current Note** — 检查当前活动笔记
   - **Current Folder** — 检查当前活动文件所在的文件夹（含子文件夹）
   - **Entire Vault** — 检查整个仓库中的所有 Markdown 文件

#### 检查内容

**1. 必填字段检查**

检查以下字段是否存在：
- `type`
- `status`
- `created`
- `updated`

> 如果文件没有 frontmatter，四个字段都会报告为缺失。

**2. 枚举值检查**

检查字段值是否在 vocabulary 允许范围内：
- `type`: 检查 type vocabulary
- `status`: 检查 status vocabulary
- `source`: 检查 source vocabulary
- `domain`: 检查 Domain vocabulary（domain 不在列表中时为 info 级别，不是 warning）

**3. 日期格式检查**

检查 `created` 和 `updated` 是否符合 `YYYY-MM-DD` 格式。

常见错误格式：
- `2026/05/09`（斜杠分隔）
- `2026-5-9`（未补零）
- `09-05-2026`（日月年顺序）

**4. 未知字段检查**

检测可能的拼写错误，例如：
- `statu` → `status`
- `tyep` → `type`
- `creatd` → `created`
- `updaetd` → `updated`
- `domian` → `domain`
- `satus` → `status`

> 未知字段检查仅提示建议，不会阻止你使用自定义字段。

#### 输出结果

检查完成后，会在仓库根目录生成 `Metadata Health Report.md` 文件并自动打开。

如果报告文件已存在，会覆盖更新。

报告示例：

```markdown
# Metadata Health Report

**Checked:** 15 file(s)
**With issues:** 3 file(s)
**Clean:** 12 file(s)

---

## Notes/Project Alpha.md

**Warnings:**

1. Missing required field: status

2. Invalid enum value: source: internet
Allowed: personal, external

3. Invalid date format: created: 2026/05/09
Expected: YYYY-MM-DD

---

## Notes/Meeting Notes.md

**Info:**

1. Domain "research" is not in predefined list: music, bigdata, aigc, ecommerce, programming, finance, system

---
```

#### 严重级别

| 级别 | 说明 | 示例 |
|------|------|------|
| `warning` | 需要关注的问题 | 缺少必填字段、枚举值不合法、日期格式错误 |
| `info` | 信息提示 | domain 不在预设列表、可能的拼写错误 |

**注意**：健康检查不会修改任何文件，仅生成报告。

## 设置选项

打开 **Obsidian 设置 → 第三方插件 → Frontmatter Bootstrap**

### Base Options

设置页面顶部是 **Base options** 分组，包含 type、status、source 三个 vocabulary 编辑器。

每个 vocabulary 编辑器显示为一个独立卡片：

```
┌─────────────────────────────────────────────────────┐
│ Type  Defines the structural role       [+ Add]    │
│                                                     │
│  note          General knowledge note            ✕  │
│  project       Project-oriented...               ✕  │
│                                                     │
│  Default type            [note ▼]                  │
└─────────────────────────────────────────────────────┘
```

#### Vocabulary 编辑

- **值列表**：显示当前所有 vocabulary 值及其描述
- **删除按钮**：点击 ✕ 删除该值（如果删除的是当前默认值，自动切换到第一个值）
- **+ Add 按钮**：位于卡片右上角，点击展开输入框添加新值
- **Default 下拉框**：从 vocabulary 中选择默认值

#### 添加新值

点击 **+ Add** 按钮后，展开输入框：
- **Value**：输入 vocabulary 值（必填）
- **Description**：输入值描述（可选，默认使用 value）
- **Save**：保存新值
- **Cancel**：取消添加

### 功能开关

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| Enable created field | 插入 `created` 字段 | ✅ 开启 |
| Enable updated field | 插入 `updated` 字段 | ✅ 开启 |
| Auto-update modified date | 保存时自动更新 `updated` 字段 | ✅ 开启 |

> 如果关闭 "Enable updated field"，模板中不会包含 `updated` 字段，自动更新也不会生效。

### Domain 选项

**Domain vocabulary**：逗号分隔的 domain 值，用于健康检查时校验。

预设 domain：
- music
- bigdata
- aigc
- ecommerce
- programming
- finance
- system

> domain 字段的值必须在 Domain vocabulary 列表中，否则健康检查会报告 warning。你可以通过设置页面修改 vocabulary 列表。

### 自定义模板

**Custom YAML template**：完全自定义插入的模板内容。

支持的变量：
- `{{date}}` — 替换为 YYYY-MM-DD 格式的日期

示例模板：
```
type: project
status: active
created: {{date}}
priority: high
```

生成结果：
```yaml
---
type: project
status: active
created: 2026-05-09
priority: high
---
```

**注意**：
- 设置自定义模板后，将**完全覆盖**默认模板，包括默认值设置
- 模板格式为 `key: value`，每行一个字段
- 空值字段会插入空字符串
- `{{date}}` 会在插入时替换为当天日期

## 安全原则

插件严格遵守以下安全原则：

1. **不扫描历史文件** — 只处理新建文件
2. **不修改旧文件** — 不会批量修改已有笔记
3. **不分析正文** — 只操作 frontmatter
4. **不自动生成标签** — tags 字段由用户填写
5. **不自动分类** — domain 由用户填写
6. **不使用 AI** — 无智能推断
7. **不联网** — 完全本地运行
8. **不做目录推断** — 不根据文件夹位置推断 metadata
9. **不覆盖已有 frontmatter** — 尊重已有内容
10. **不重复插入 metadata** — 检测已有 frontmatter

## 常见问题

### Q: 为什么我的新笔记没有自动插入模板？

检查以下条件：
1. 文件是否为 `.md` 扩展名
2. 文件是否完全为空（无任何字符，包括空格和换行）
3. 是否已有 frontmatter（以 `---` 开头）

> 使用模板创建的笔记可能已有内容，不会触发自动插入。

### Q: 如何修改已插入的模板？

插件只在新文件创建时插入模板，已创建的文件不会被修改。你可以：
1. 手动编辑 frontmatter
2. 使用 Obsidian 的 Properties 面板
3. 使用 Dataview 批量查询和定位需要修改的笔记

### Q: updated 字段什么时候更新？

每次文件保存时检查，如果 `updated` 字段存在且日期不是今天，则更新为今天。内置 500ms 防抖，避免频繁保存时重复写入。

### Q: Health Check 会修改我的文件吗？

不会。Health Check 只生成报告文件（`Metadata Health Report.md`），不会修改任何笔记文件。

### Q: 如何添加自定义 type/status/source 值？

在设置页面的 **Base options** 区域，找到对应的 vocabulary 卡片，点击右上角的 **+ Add** 按钮，输入 Value 和 Description 后保存。

### Q: 如何添加自定义 domain？

在设置页面的 "Domain vocabulary" 文本框中，用逗号分隔输入 domain 值。你可以修改、添加或删除列表中的值。

### Q: 自定义模板和默认值设置有什么关系？

设置自定义模板后，默认值设置（Default type、Default status、Default source）将不再生效。自定义模板完全控制插入的内容。

### Q: domain 字段是枚举值吗？

domain 字段必须在 Domain vocabulary 列表中，否则健康检查会报告 warning。你可以通过设置页面修改 vocabulary 列表来添加或删除允许的 domain 值。

### Q: 插件会在我不知情的情况下修改文件吗？

不会。插件只做两件事：
1. 新建空文件时插入模板（可预测、可配置）
2. 保存时更新 `updated` 字段（可关闭）

所有操作都是确定性的，没有隐藏行为。
