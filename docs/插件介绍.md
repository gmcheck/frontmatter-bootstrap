# Frontmatter Bootstrap — 极简的 Obsidian Metadata 基础设施插件

## 这是什么？

**Frontmatter Bootstrap** 是一个 Obsidian 插件，专注于做一件事：**让你的笔记拥有统一、稳定的 metadata 结构**。

创建新笔记时，它会自动插入这样的 YAML frontmatter：

```yaml
---
type: note
domain:
status: active
created: 2026-05-09
updated: 2026-05-09
source: personal
tags:
---
```

就这么简单。没有 AI，没有智能分类，没有自动标签推断。

## 为什么需要它？

如果你使用 Obsidian 一段时间，可能会发现：

- 每个笔记的 frontmatter 格式不一致
- 有些笔记有 `created`，有些没有
- `status` 字段有时候写 `done`，有时候写 `finished`
- 想用 Dataview 查询，但字段名不统一
- 日期格式五花八门，有的用 `/`，有的用 `-`

Frontmatter Bootstrap 解决的就是这个问题：**为你的笔记提供一致的 metadata 基础**。

一旦你的笔记有了统一的 metadata 结构，就可以：

- 用 **Dataview** 精确查询特定类型、状态的笔记
- 用 **Bases** 构建结构化视图
- 用 **Obsidian Properties** 面板高效编辑
- 用 **搜索** 按字段精确过滤

## 三个核心功能

### 1. 自动插入模板

创建新的空 Markdown 文件时，自动插入预定义的 frontmatter 模板。

**触发条件**（必须全部满足）：
- 文件是 `.md` 扩展名
- 文件内容完全为空
- 文件没有已有 frontmatter

只有新建的空文件才会触发，已有文件永远不会被修改。

### 2. 自动更新日期

每次保存文件时，自动将 `updated` 字段更新为当天日期。

- 只更新 `updated`，不会动 `created`
- 只操作 frontmatter，不会修改正文
- 如果文件没有 `updated` 字段，不会添加

### 3. Metadata 健康检查

手动执行检查，发现已有笔记中的 metadata 问题：

- **缺少必填字段** — type、status、created、updated 缺失
- **枚举值不合法** — type 写了不在列表中的值
- **日期格式错误** — created/updated 不是 YYYY-MM-DD
- **可能的拼写错误** — `statu` → `status`，`tyep` → `type`

检查结果生成报告文件，不会修改任何笔记。

## 设计哲学

这个插件的设计原则是：

- **极简** — 只做必要的事
- **稳定** — 长期可用，不会频繁变动
- **低侵入** — 不干扰你的工作流
- **可预测** — 行为明确，没有意外
- **确定性** — 不做语义理解或智能分析

## 它不是什么

- ❌ 不是 AI 插件
- ❌ 不是自动分类工具
- ❌ 不是知识管理系统
- ❌ 不会推断你的笔记属于哪个领域
- ❌ 不会自动生成标签
- ❌ 不会后台扫描你的文件
- ❌ 不会联网

这些事情留给你自己决定。

## 快速开始

1. 在 Obsidian 社区插件市场搜索 **"Frontmatter Bootstrap"**
2. 安装并启用
3. 创建一个新的 Markdown 文件
4. 完成 — frontmatter 已自动插入

## 自定义

你可以在设置中：

- **修改默认值** — type、status、source 的默认值
- **配置 domain vocabulary** — 逗号分隔，定义合法 domain 值
- **完全自定义模板** — 用 `key: value` 格式定义模板，支持 `{{date}}` 变量
- **开关功能模块** — 启用/禁用 created、updated 字段和自动更新日期

### 自定义模板示例

在设置中输入：

```
type: project
status: active
created: {{date}}
priority: high
```

新建笔记时就会插入：

```yaml
---
type: project
status: active
created: 2026-05-09
priority: high
---
```

## 与 Dataview 配合

统一的 metadata 结构让 Dataview 查询变得简单可靠：

```dataview
TABLE type, status, created
WHERE type = "project" AND status = "active"
SORT created DESC
```

```dataview
LIST
WHERE domain = "programming"
```

## 兼容性

- Obsidian Properties ✓

## 总结

如果你想要一个简单、稳定、不折腾的 metadata 基础设施，Frontmatter Bootstrap 可能适合你。

它不会改变你的工作方式，只是让你的笔记更一致、更规范。
