```text id="jlwm_final"
开发一个 Obsidian 插件：

frontmatter-bootstrap

========================================
一、插件定位（核心）
========================================

这是一个：

“长期主义 frontmatter 基础设施插件”

插件目标：

1. 新建 Markdown 笔记时：
自动插入统一 YAML metadata 模板

2. （暂时不实现这部分功能）自动初始化 Obsidian Property Schema：
让固定字段具备：
- Select
- Date
- Tags

Obsidian 暂时对这部分支持能力有限，这部分等 Properties UI 能力。

3. 提供：
轻量级 Schema Validation（手动健康检查）

========================================
二、核心设计哲学（必须遵守）
========================================

插件必须：

- 极简
- 长期稳定
- 低侵入
- 不做 AI
- 不自动分类
- 不自动生成标签
- 不分析正文
- 不后台扫描
- 不实时校验

----------------------------------------

插件目标：

不是：
“智能知识系统”

而是：

“长期稳定 metadata 基础设施”

========================================
三、Metadata Bootstrap 功能
========================================

当用户创建新的 Markdown 文件时：

如果：
- 文件为空
- 没有 frontmatter

则：

自动插入：

---
type: note
domain:
status: active
created: 2026-05-09
updated: 2026-05-09
source: personal
tags:
---

----------------------------------------

日期格式：

YYYY-MM-DD

========================================
四、最终 Metadata Schema（固定）
========================================

----------------------------------------
1. type
----------------------------------------

字段类型：
Select

允许值：

- note
- project
- archive

默认：

note

----------------------------------------
2. domain
----------------------------------------

字段类型：
Select

默认值：

- music
- bigdata
- aigc
- ecommerce
- programming
- finance
- system

允许用户扩展。

插件不要自动推断 domain。

----------------------------------------
3. status
----------------------------------------

字段类型：
Select

允许值：

- active
- done
- archived

默认：

active

----------------------------------------
4. created
----------------------------------------

字段类型：
Date

格式：

YYYY-MM-DD

仅创建时写入一次。

----------------------------------------
5. updated
----------------------------------------

字段类型：
Date

格式：

YYYY-MM-DD

文件保存时自动更新。

----------------------------------------
6. source
----------------------------------------

字段类型：
Select

允许值：

- personal
- external
- ai

默认：

personal

----------------------------------------
7. tags
----------------------------------------

字段类型：
Tags

========================================
五、Property Schema Initialization
========================================

插件首次启动时：

自动初始化：

Vault Property Definitions

目标：

让固定字段自动拥有：

- Select 下拉
- Date
- Tags

等 Obsidian Properties UI 能力。

========================================
六、Property Definitions（正式版）
========================================

----------------------------------------
type
----------------------------------------

Type:
Select

Options:

- note
- project
- archive

----------------------------------------
domain
----------------------------------------

Type:
Select

Options:

- music
- bigdata
- aigc
- ecommerce
- programming
- finance
- system

----------------------------------------
status
----------------------------------------

Type:
Select

Options:

- active
- done
- archived

----------------------------------------
source
----------------------------------------

Type:
Select

Options:

- personal
- external
- ai

----------------------------------------
created
----------------------------------------

Type:
Date

----------------------------------------
updated
----------------------------------------

Type:
Date

----------------------------------------
tags
----------------------------------------

Type:
Tags

========================================
七、updated 自动更新时间
========================================

当 Markdown 文件保存时：

如果：
存在：

updated:

字段。

则：

自动更新：

updated: YYYY-MM-DD

----------------------------------------

要求：

1. 仅更新 updated
2. 不修改 created
3. 不修改其他字段
4. 不覆盖 frontmatter
5. 不修改正文

========================================
八、Schema Validation（最终版）
========================================

Validation：

是：

“Metadata Health Check”

而不是：

实时约束系统。

----------------------------------------

Validation：

仅允许：

用户手动执行。

========================================
九、Validation 功能（统一设计）
========================================

不要拆分：

- Validate Current Note
- Validate Folder
- Validate Vault

多个命令。

----------------------------------------

改为：

一个统一入口。

========================================
十、最终 Validation 命令（正式版）
========================================

Command Palette：

Metadata Health Check

========================================
十一、Metadata Health Check 执行逻辑
========================================

执行命令后：

弹出选择菜单。

用户选择：

----------------------------------------
1. Current Note
----------------------------------------

检查当前 Markdown 文件。

----------------------------------------
2. Current Folder
----------------------------------------

检查当前目录。

----------------------------------------
3. Entire Vault
----------------------------------------

检查整个 Vault。

========================================
十二、Validation 检查内容
========================================

----------------------------------------
1. Required Field Validation
----------------------------------------

检查：

- type
- status
- created
- updated

是否存在。

----------------------------------------
2. Enum Validation
----------------------------------------

检查：

----------------------------------------
type
----------------------------------------

允许：

- note
- project
- archive

----------------------------------------
status
----------------------------------------

允许：

- active
- done
- archived

----------------------------------------
source
----------------------------------------

允许：

- personal
- external
- ai

----------------------------------------
domain
----------------------------------------

允许默认：

- music
- bigdata
- aigc
- ecommerce
- programming
- finance
- system

----------------------------------------
3. Date Format Validation
----------------------------------------

检查：

created
updated

是否符合：

YYYY-MM-DD

----------------------------------------
4. Unknown Field Validation（可选）
----------------------------------------

检查：

可能 typo 字段。

例如：

statu → status

========================================
十三、Validation 输出方式
========================================

不要：

- 实时弹窗
- 实时 Warning
- 后台 Validation
- 阻塞 UI

========================================
十四、Validation Report（正式版）
========================================

Validation 执行完成后：

生成：

Metadata Health Report

----------------------------------------

推荐形式：

Markdown Report

========================================
十五、Validation Report 示例
========================================

Metadata Health Report

File:
Knowledge/Music/Jazz.md

Warnings:

1. Missing required field:
status

2. Invalid enum value:
source: internet

Allowed:
personal, external, ai

3. Invalid date format:
created: 2026/05/09

Expected:
YYYY-MM-DD

========================================
十六、Validation 严重级别
========================================

仅支持：

- warning
- info

----------------------------------------

不要：

Error Blocking

========================================
十七、技术实现要求
========================================

允许：

使用：

app.metadataTypeManager

初始化 Property Definitions。

----------------------------------------

要求：

1. 幂等执行
2. 不覆盖用户已有配置
3. 不破坏已有 schema

========================================
十八、Frontmatter 操作要求
========================================

必须使用：

processFrontMatter()

或：

安全 YAML API。

----------------------------------------

禁止：

正则暴力替换 frontmatter。

========================================
十九、项目结构（正式版）
========================================

src/

    template.ts
    metadata.ts

    types.ts

二十、模块职责
========================================

插件入口。

----------------------------------------

----------------------------------------

metadata 模板生成。

----------------------------------------

----------------------------------------

Property Schema 初始化。

----------------------------------------

validation/engine.ts

Validation Engine。

----------------------------------------

validation/report.ts

生成 Metadata Health Report。

----------------------------------------

validators/*

各类 validation 检查。

========================================
二十一、性能要求（非常重要）
========================================

插件必须：

1. 不全库后台扫描
2. 不实时 Validation
3. 不影响启动速度
4. 不增加大型依赖
5. 不监听无关事件

----------------------------------------

Validation：

仅在用户主动执行时运行。

========================================
二十二、明确禁止实现的功能
========================================

不要实现：

- AI 分类
- 自动 domain 推断
- 自动 tags
- embedding
- graph analysis
- OCR
- 云同步
- 向量数据库
- 自动摘要
- 实时 Validation
- 自动修复

========================================
二十三、最终设计目标
========================================

这个插件：

不是：

“智能 PKM 平台”

而是：

一个：

- 极简
- 稳定
- 长期主义
- 低维护
- 几乎无感存在

的 metadata 基础设施插件。

========================================
二十四、最终输出要求
========================================

请直接生成：

1. 完整插件代码
2. manifest.json
3. package.json
4. tsconfig.json
5. 所有 TypeScript 文件
6. README
7. 可直接 npm install / build 的完整工程

确保：

符合 Obsidian 社区插件规范，
并可以直接运行。
```
